# Active NPC Runtime Template

用于 `state.md -> Onstage NPCs` 区块。
只记录当前运行态，不写长篇人物小传。

```md
- 代称：
- 真名解锁：未解锁 / 已解锁（若已解锁，写明当前可用称呼）
- 身份标签：
- 当前位置：
- 当前状态：
- 当前重要性：高 / 中 / 低
- 对主角当前态度：
- 当前目标 / 当前顾虑：
- 当前已知关键信息：
- 当前误判 / 不确定点：
- 当前持有关键物件：
- 是否参与当前事件线：是 / 否
- 最近公开行动：
- 最近私下行动：
```

## 填写规则
- `身份标签` 只写当前场景需要的最小识别信息，例如“灰衣剑客 / 镇北司缇骑 / 镖师 / 客栈掌柜”。
- `当前位置` 需要能支持下一轮裁定；若不明，明确写“不明”，不要脑补。
- `当前状态` 只写运行态，例如“轻伤 / 警惕 / 潜伏中 / 与人同行 / 失联”。
- `当前重要性` 用于热上下文裁剪。默认只把真正影响下一轮的人标成“高”；临时角色、边缘角色或已离场角色应尽量维持“中/低”。
- `当前已知关键信息` 只写此 NPC 真的有来源知道的内容。
- `当前误判 / 不确定点` 很重要，用来避免 NPC 全知。
- `最近公开行动` 与 `最近私下行动` 用于区分公开情报和私密信息，不必每轮都写满。
- 若某 NPC 不再活跃，应从 `state.md` 移除或压缩，不在这里长期堆积。

---

补充说明：

- `Onstage NPCs` 只放“当前在场、下一轮立刻能互动”的人物。
- 若某角色当前未必在场，但仍然直接牵动局势，应放到 `Relevant NPCs`，不要继续占着 `Onstage NPCs`。
