# Protagonist Profile Layering

主角建议分两层维护，不要混写。

## 长期层
- 文件：`player-profile.md` / `player-profile.json`
- 记录内容：姓名、出身、外貌、性格、喜好、背景、长期能力与心理底色。
- 用途：保证主角固定设定稳定，不随单轮剧情漂移。

## 运行层
- 文件：`state.md -> Protagonist Runtime`
- 记录内容：当前位置、伤势、暴露点、同行者、当前压力、关键物件、公开/私密信息。
- 用途：保证下一轮续写和裁定不出错。

## 原则
- 不在 `state.md` 重复主角的长期人物介绍。
- 不在 `player-profile` 里堆当前场景细节。
- 若某次经历已经沉淀成长期影响，再从 `summary` 或剧情中提炼后回写到 `player-profile` 的合适位置。
