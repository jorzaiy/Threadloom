# State

## Usage Rules
- 这里只记录运行时状态，不写长篇人物小传。
- 只保留“下一轮看不到就可能演错”的事实。
- 长期人物性格、过往经历、稳定立场与关系史，放到 `memory/npcs/` 或其他长期记忆层，不挤进这里。
- 新登场 NPC 默认只写当前可确认状态，不补全未经确认的背景。
- 私下发生的信息，不因进入历史就自动变成公开信息。

## World Time
- 当前时间：承和十二年，三月初七，入夜。
- 最近一次明确时间推进：入夜。
- 时间原则：涉及赶路、等待、养伤、休息、搜查、跟踪、谈判、转场时，都要明确写出时间如何前进。

## Current Scene
- 当前地点：神都相关场景。
- 当前场景：基于最近同步会话自动提取，需人工复核。
- 当前局势核心：局势仍在流动；最近场景中的风险、人物关系与异常动静。

## Update Policy
- `state` 只保留当前运行态，不承担长期设定存档功能。
- 当前脚本是通用启发式版本，适合先跑通，但高质量状态整理后续最好接入受约束 LLM。
- 需要长期记住但不属于运行时的信息，应转写到 `summary`、`canon` 或 NPC 档案。
- 若某信息已不再影响下一轮演出，应及时从这里移除或压缩。
